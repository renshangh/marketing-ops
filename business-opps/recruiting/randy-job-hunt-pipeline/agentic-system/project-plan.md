# Recruiting Project Plan

## Goal
Build a recruiting AI operating system starting from the Randy Job Hunt Pipeline, then generalize the workflow into a broader recruiting coordination platform.

## Working Folder
All recruiting work should live inside:
- `business-opps/recruiting/`

## Primary Objective
Create a system that helps manage hiring opportunities, candidate workflow, company posting, follow-up, and coordination through a mix of structured data and agent assistance.

---

## Workstreams

### 1. Product Strategy
Define the user-facing problem, market wedge, and long-term direction.

### 2. Architectural Design
Define the overall system structure, major components, and how the pieces fit together.

### 3. Data Flow Design
Define how opportunity data, application data, and workflow state move through the system.

### 4. Agentic AI Design
Define where agents operate, what they do, and where human control remains.

### 5. Implementation Planning
Sequence the build so the system becomes useful early and generalizes later.

---

## Deliverables

### Required planning outputs
- Project plan
- Overall architecture overview
- Overall design diagram
- Sequential data diagram
- Agentic AI system design diagram

---

## Phased Build Direction

### Phase 1
Candidate-side pipeline and structured opportunity tracking.

### Phase 2
Agent-assisted coordination and workflow management.

### Phase 3
Generalizable recruiting operations abstractions.

### Phase 4
Broader recruiting coordination platform expansion.

---

## Success Criteria
- The system is not just a job tracker
- The workflow is structured and reusable
- Agent tasks are clear and bounded
- Data flow is explicit
- The design can expand beyond the first user

---

## Key Principle
Build for a real workflow first, then generalize the abstractions.
